# Adding events (agent guide)

Instructions for adding a robotics event to this dataset. The canonical source is `events.yaml`; do not edit `README.md` manually — it is regenerated by CI (`scripts/autogenerate_readme.py` via `.github/workflows/recreate_readme.yaml`).

## Where events appear

- **Weekly Robotics site:** `_data/robotic_events/events.yaml` is loaded by Jekyll as `site.data.robotic_events.events` and rendered on [weeklyrobotics.com/events](https://www.weeklyrobotics.com/events) via `_includes/events.html`.
- **This repo's README:** autogenerated from the same YAML, grouped by category.

Past events live in `past_events.yaml` (moved there automatically by `scripts/clean_events.py` / `.github/workflows/daily_prune.yaml` once `start_date` or `end_date` is in the past).

## Event schema

Each entry in `events.yaml` is a YAML list item:

```yaml
- title: Physical AI Happy Hour: Zürich
  link: https://luma.com/qa79tj3g
  start_date: 2026-07-16
  city: Zurich
  country: Switzerland
  category: Meetup
```

| Field | Required | Notes |
|-------|----------|-------|
| `title` | yes | Use the event's official name. Quote the value if it contains `:`, `#`, or other YAML-special characters. |
| `link` | yes | Registration or info URL (event page, Luma, etc.). |
| `start_date` | yes | ISO date: `YYYY-MM-DD`. |
| `end_date` | no | Omit for single-day events. |
| `city` | yes | City name; include state/region when helpful (e.g. `San Francisco, California`). |
| `country` | yes | Full country name (e.g. `Switzerland`, `United States of America`) — match existing entries. |
| `category` | yes | One of: `Conference`, `Expo`, `Workshop`, `Competition`, `Meetup`. |
| `deadline` | no | Optional submission/registration deadline (`YYYY-MM-DD`). |

**Do not add** events related to weaponry or weaponization.

## How to gather details

1. Open the event URL (Luma, conference site, Meetup, etc.).
2. Extract: title, link, date(s), location (city + country), and pick the best category.
3. For Luma events, the page usually shows title, date/time, and neighborhood or city; exact address may be hidden until registration — use the public city (e.g. Oerlikon → `Zurich`).
4. Search `events.yaml` and `past_events.yaml` for the same `link` to avoid duplicates.

## Insertion rules

1. Edit **only** `events.yaml`.
2. Insert the new block in **chronological order** by `start_date` (earliest first).
3. Keep YAML indentation consistent (2 spaces per level).
4. Do **not** edit `README.md` — CI updates it on push to `main`.

## Category hints

| Category | Typical events |
|----------|----------------|
| `Conference` | Multi-day academic or industry conferences |
| `Expo` | Trade shows, expos |
| `Workshop` | Hands-on or tutorial workshops |
| `Competition` | Robotic competitions |
| `Meetup` | Happy hours, local meetups, single-evening gatherings |

When unsure between `Meetup` and `Workshop`, prefer `Meetup` for informal social/networking events.

## Checklist before finishing

- [ ] Event is in `events.yaml` at the correct chronological position
- [ ] `link` is not already present in `events.yaml` or `past_events.yaml`
- [ ] Dates are `YYYY-MM-DD`
- [ ] `category` is one of the five allowed values
- [ ] Title is quoted if it contains `:`
- [ ] `README.md` was **not** edited manually
